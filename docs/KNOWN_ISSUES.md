# 既知の不具合

## URLの気温種別を指定して再読み込みするとハイドレーションエラーになる

### 状況

未修正。ポートフォリオ改善作業中のブラウザ検証で確認した。

### 再現手順

1. `?pref=44&station=4&type=min` のように、`type` に `max` 以外を指定して画面を開く
2. ページを再読み込みする
3. ブラウザコンソールを確認する

Reactのハイドレーションエラー（production buildではReact error #418）が出力される。

### 原因

`frontend/src/hooks/use-url-params.ts` の `parseParams()` が、サーバーレンダリング時には
既定値の `type: "max"` を返し、ブラウザの初回レンダーではURLから `type: "min"` または
`type: "avg"` を返す。この値を `frontend/src/app/page.tsx` のstate初期値と表示テキストに
使用しているため、サーバーHTMLとブラウザの初回描画が一致しない。

### 影響

- `type=min` / `type=avg` を含むURLの再読み込み時にハイドレーションエラーが記録される
- Reactによるクライアント側の再描画が発生する
- 通常操作による気温種別の切り替え自体は動作する

### 修正方針案

- URLを外部ストアとして扱い、サーバー用スナップショットとブラウザ用スナップショットを分離する
- サーバーHTMLとブラウザ初回描画では同じ既定値を使用し、URLの復元はハイドレーション後に行う
- 都道府県・地点・気温種別をURLと同期する既存動作、および共有URLからの復元を維持する
- `type=max` / `type=min` / `type=avg` の各URLについて、再読み込み時のコンソールエラーがないことをブラウザテストで確認する
